# Acutework

> Safety alerts for medical practitioners. Verified for IMA & state councils.

A production-grade Flutter application supporting **Android** and **iOS**, built with
Clean Architecture, Riverpod, go_router, Dio + Retrofit, and Hive.

## ✨ Stack

| Concern             | Choice                                              |
| ------------------- | --------------------------------------------------- |
| Architecture        | Clean Architecture (data / domain / presentation)   |
| State management    | Riverpod 2 (+ riverpod_generator)                   |
| Routing             | go_router                                           |
| Networking          | Dio + Retrofit                                      |
| Local storage       | Hive + flutter_secure_storage                       |
| Models              | freezed + json_serializable                         |
| Dependency injection| get_it + Riverpod providers                         |
| Notifications       | Firebase Messaging + flutter_local_notifications    |
| Localization        | flutter_localizations + intl                        |
| Lints               | very_good_analysis                                  |
| Flavors             | dev / staging / prod                                |

## 🎨 Design system

Acutework · deep teal + clinical white · Material 3 + iOS sensitive.

| Token           | Hex        | Usage                |
| --------------- | ---------- | -------------------- |
| Primary teal    | `#0F766E`  | Trust, default       |
| Deep teal       | `#0A4F4A`  | Hero gradients       |
| Safe green      | `#4F9A6B`  | Mark safe / OK       |
| Warn amber      | `#E5A24B`  | Setup needed         |
| Emergency red   | `#E5414B`  | Alert action ONLY    |
| Ink             | `#0B1416`  | Body text            |
| Muted           | `#6B7280`  | Secondary text       |
| Hairline        | `#E5E7EB`  | Borders              |

Typography: **Inter** — Display 32/700, Title 22/700, Body 15/600 strong, Body 13.5/400, Caption 11 uppercase.

**Voice & tone:** Reassuring, professional, direct. Red is *only* for emergency action — never decoration.

## 🛠️ First-time setup

Do this **once** after cloning the repo. Subsequent runs only need the steps in [Daily run](#-daily-run).

### 1. Install Flutter

```bash
flutter --version
```

If missing:

```bash
brew install --cask flutter        # macOS, easiest
# or download from https://docs.flutter.dev/get-started/install/macos
flutter doctor                     # resolve any ❌ items before continuing
```

### 2. Generate the native Android & iOS projects

The repo only ships Dart code — native folders are generated by Flutter:

```bash
cd path/to/shield
rm -f android/.gitkeep ios/.gitkeep
flutter create . --platforms=android,ios --org in.acutework --project-name acutework
```

### 3. Add the Inter font files

Download from [rsms.me/inter](https://rsms.me/inter/) and place these in `assets/fonts/`:

```
Inter-Regular.ttf  Inter-Medium.ttf  Inter-SemiBold.ttf  Inter-Bold.ttf
```

> Or temporarily comment out the `fonts:` block in `pubspec.yaml` if you want to skip.

### 4. Create the env file

```bash
cp .env.example .env
```

### 5. Install dependencies & run codegen

```bash
flutter pub get
dart run build_runner build --delete-conflicting-outputs
```

### 6. iOS-specific setup (macOS only)

```bash
sudo gem install cocoapods            # one-time, if not installed
cd ios && pod install && cd ..
open -a Simulator                     # boot the iOS Simulator
```

In Xcode: open `ios/Runner.xcworkspace`, set your **Team** under
*Runner → Signing & Capabilities*, and update the bundle identifier if needed.

### 7. Android-specific setup

- Install **Android Studio** + the latest Android SDK + an Android emulator image.
- Accept SDK licenses:

  ```bash
  flutter doctor --android-licenses
  ```

- Launch an emulator (or plug in a device with USB debugging on):

  ```bash
  flutter emulators                       # list
  flutter emulators --launch <id>         # boot one
  ```

### 8. (Optional) Configure flavors

Flavor builds (`--flavor dev/staging/prod`) require native config — see
[`PLATFORMS.md`](./PLATFORMS.md). Until that's done, just use plain `flutter run`
which uses `lib/main.dart` → dev flavor.

### 9. Verify

```bash
flutter doctor -v
flutter devices
flutter analyze
flutter test
```

## 🚀 Daily run

Once first-time setup is done, day-to-day commands are:

```bash
flutter pub get                        # only after pulling new dependencies
flutter run                            # default → dev flavor on selected device
```

### Pick a device explicitly

```bash
flutter devices                        # list connected/available
flutter run -d <device_id>             # e.g. -d "iPhone 15 Pro" or -d emulator-5554
```

### Run a specific flavor (after flavor setup)

```bash
./scripts/run.sh dev                   # or staging / prod
# equivalent to:
flutter run --flavor dev -t lib/main_dev.dart
```

### Hot reload / restart while running

| Key | Action          |
| --- | --------------- |
| `r` | Hot reload      |
| `R` | Hot restart     |
| `q` | Quit            |
| `p` | Toggle paint baselines |
| `o` | Toggle platform (iOS / Android) |

### Re-run codegen (after editing freezed / json / retrofit / riverpod / hive sources)

```bash
dart run build_runner watch --delete-conflicting-outputs   # watcher
# or one-shot:
dart run build_runner build --delete-conflicting-outputs
```

### From VS Code

Use `Run & Debug` → pick **Acutework (dev / staging / prod / profile mode)** —
launch configs are pre-wired in `.vscode/launch.json`.

### Lint & format

```bash
dart format .
flutter analyze
```

### Clean build (when things misbehave)

```bash
flutter clean && flutter pub get
cd ios && pod install --repo-update && cd ..   # iOS only
```

## 🧪 Testing

```bash
flutter test                      # unit + widget
flutter test integration_test/    # integration
flutter test --coverage
```

flutter build apk    --flavor prod -t lib/main_prod.dart --release
